Olá, sejam bem vindos. Nesta página é apresentado o projeto que foi desenvolvido para o Trabalho de Conclusão de Curso no ano de 2016.



Proposta

A ideia desse Projeto é ter um arcabouço de ecossistema virtual, ter 3 coisas distintas que interajam entre si. Essas três coisas são:

  • Um aplicativo de celular que envia e recebe dados de um servidor;
  • Uma página Web que tenha algum funcinamento que possa ser atualizado com dados vindos de um servidor;
  • Um servidor que faça essa ligação entre o aplicativo e a página, mandando e recebendo dados de ambos.

Essa estrutura pronta serve de base para projetos maiores. Exemplos de uso incluem:

  • Um jogo executado no browser, que utiliza o celular como controle.
  • Um aplicativo de buscas em um Banco de Dados, no qual esse Banco fique localizado em um servidor para aproveitar o poder computacional ou espaço que o celular não possui.



Em um exemplo mais concreto, o que foi implementado tem o seguinte funcionamento básico:

  • O aplicativo envia dados de seu acelerômetro para o servidor;
  • O servidor encaminha esses dados para a página Web;
  • A página exibe uma esfera que utiliza esses dados para alterar sua velocidade de rotação nos eixos X, Y e Z.



Eu passei um ano tentando achar uma maneira simples e funcional de explicar essa proposta, mas, infelizmente não encontrei nenhuma maneira formal. A maneira mais concreta de explicar que eu encontrei foi:

  • Você inclina o seu celular e a bola na tela do computador começa a rodar.


É mais fácil de entender, não?

Dificuldades e desafios

1. Desafios e frustrações encontrados

Aqui é pra chorar as pitangas? Vou chorar as pitangas.

Aspectos da implementação:

O primeiro ponto foi o desafio da tecnologia necessária para o Projeto. Eu não tinha experiência em nenhum dos pontos propostos. Nunca tinha feito um aplicativo para Android do zero, nunca tinha mexido com servidores e possuia uma experiência mínima com a construção de sites. Era necessário entender vários conceitos, aprender várias tecnologias e bater muito a cabeça.

E bater a cabeça foi uma coisa que eu fiz muito. Até chegar ao código final, com as tecnologias e ferramentas certas para o funcionamento, eu tentei várias outras coisas. Cada passo de progresso envolveu no mínimo três batidas de cabeça.


Para criar o aplicativo, inicialmente eu tentei utilizar principalmente o Kivy, um framework de Python para fazer Apps. Inicialmente parecia promissor, mas eu tive muitos problemas para fazer a conexão com o servidor usando Sockets.

Para deixar mais claro: eu tentei fazer essa conexão de várias maneiras, utilizando muitas tecnicas e lendo muitos exemplos, mas eu não consegui de jeito nenhum resolver os problemas que apareciam. Em lugar nenhum na Internet eu achava soluções.

Uma hora eu desisti do Kivy e de Python, para tentar utilizar JavaScript e ter uma única linguagem utilizada no código (me confundia muito ter que utilizar duas linguagens no mesmo projeto, principalmente porque eu não tinha experiência em nenhuma das duas), e fui para o Cordova.

Com o Apache Cordova eu tive muito mais sucesso, a conexão me deu muito menos dores de cabeça. Um dos problemas encontrados foi a cláusula de segurança que ele oferece para conexões com servidores. Ela impedia a conexão com o Socket do servidor e eu não achava em lugar nenhum uma documentação boa para fazê-la funcionar.

Eventualmente foi necessário remover completamente essa cláusula e reduzir o escopo do projeto para conexões locais, evitando perigos de segurança. Uma parte da segurança é assegurada pelo protocolo de Sockets, então foi possível tomar essa decisão.

Usar sockets foi outra decisão necessária que demorou para sair. O problema surgiu quando eu tentei enviar dados do servidor para o cliente, sem sucesso. O protocolo HTTP, que eu estava utilizando, não permite esse fluxo, mas eu não sabia e tentei de várias maneiras. Uma hora eu cheguei na API Socket.io para JavaScript que resolvia esse problema, mas foi necessário entender alguns pontos para conseguir fazer sua implementação.

A última parte de implementação foi a criação da página Web 3D. Desde o início eu tinha esse planejamento de fazer algo 3D, mas o escopo era maior no início, eu pretendia criar um gerador procedural de terrenos. Encontrar a biblioteca 3D THREE.JS simples, e sua implementação foi direta, não houve mais problemas do que o esperado. A maior dificuldade foi para criar o gerador e utilizar dados externos para alterar a geração. Eu fiz uma implementação básica e ela está no código, mas resolvi não apresentá-la, uma vez que eu só peguei o algoritmo da internet, fiz o mínimo de alterações.



Aspectos pessoais:

O ano de 2016 foi bastante corrido para a produção desse TCC. O primeiro fato é que, desde Outubro de 2015 eu estava trabalhando como estagiário, fazendo 30 horas por semana. Durante os dois semestres, meu tempo livre era muito curto, eu tinha somente as noites durante a semana e os fins de semana para dedicar ao Projeto.

O melhor momento para avançar no TCC seria o período de férias do IME, no qual eu só teria o estágio para me tomar tempo. O problema foi que, logo no início de Junho eu precisei ser internado. Meu período no hospital foi curto, mas fiquei impossibitado de digitar por quase 2 meses.

No início do segundo semestre eu estava bem recuperado já, porém muito atrasado. Outro problema é que ainda faltavam duas matérias que exigiam muito tempo de estudo.

Bateu o desespero.

O fato que mudou tudo foi que, como em Outubro eu completaria um ano de estágio, eu poderia tirar 20 dias de férias. Então elas foram programadas para ainda incluirem uma semana de break do IME.

Esse período de férias foi o que salvou o meu TCC. Eu as aproveitei ao máximo, consegui tirar todo o atraso e terminar toda a implementação.

Ainda foi bastante difícil, mas após as férias foi possível conciliar as matérias, o estágio e a Monografia.

2. Aprendizados

Como já explicado acima, meu TCC me exigiu uma gama de conceitos que eu não estava familiarizado. Olhando em retrospecto, eu não possuia conhecimento em nenhum ponto de implementação ou teoria aplicado. Vou listar cada ponto que eu precisei entender para esse projeto:

  • Desenvolvimento Android (e Mobile no geral)
  • Desenvolvimento Web
  • Desenvolvimento de um Servidor
  • Modelo cliente-servidor
  • Protocolo HTTP
  • Protocolo WebSocket
  • Computação Gráfica
  • Algoritmos de ruído
  • Node.js
  • Framework Express.js
  • Framework Socket.io
  • Framework Kivy
  • Framework Apache Cordova
  • Framework THREE.js

3. Futuro

Esse projeto, por design, serve de base para projetos maiores que utilizam essa arquitetura. Agora que sua implementação está pronta, é possível pensar em casos de uso para explorar essas tecnologias.

Aprimoramentos do código incluem tornar seu código mais limpo e didático, para facilitar o uso por outras pessoas; aumentar o escopo de conexão, de uma rede local para global; implementar mais ferramentas, como sensores de Android ou mais bibliotecas Web, etc.

Esse projeto pode crescer arbitrariamente, ele não é fechado nem feito para um uso específico.
